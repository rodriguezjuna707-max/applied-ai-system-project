UPGRADE PLAN: Music Recommender -> AI-Powered RAG System
=========================================================

ADVANCED AI FEATURE: RAG + Claude API
--------------------------------------
The existing score_song() / recommend_songs() becomes the RETRIEVAL layer.
Claude API becomes the GENERATION layer -- it receives the scored songs as
context and generates a personalized natural language explanation.
This is true RAG, fully integrated into the main application logic.

PHASE 2 -- Add Claude RAG Module
---------------------------------
New file: src/rag_explainer.py

Single public function:
    explain_with_claude(user_prefs: dict, top_songs: list, query: str) -> str

Flow:
  1. Retrieval already done -- top_songs arrives pre-scored from recommend_songs()
  2. Build a structured context block: title, artist, score, reasons for each song
  3. Send to Claude with the context injected into the prompt -- Claude reasons over it
  4. Return Claude's natural language explanation
  5. On API error -> fall back to the rule-based reasons list (logged as WARNING)
  6. Use prompt caching on the system message (cost optimization)

Claude model: claude-sonnet-4-6
Max tokens: 512
Client init: anthropic.Anthropic() (reads ANTHROPIC_API_KEY from environment)


PHASE 3 -- Logging & Guardrails
---------------------------------
New file: src/logger_config.py
  - setup_logging(log_file="recommender.log") function
  - Console handler: WARNING+ only
  - File handler (recommender.log): DEBUG+ everything
  - Format: "timestamp | module | level | message"
  - Called once at startup in app.py and main.py

Guardrails to add:
  - Empty song list -> return canned message, skip API call (log WARNING)
  - k <= 0 in Recommender.recommend() -> raise ValueError
  - Claude returns empty/short response -> fall back to rule-based (log WARNING)
  - CSV file missing -> raise descriptive error with clear message
  - Truncate context to top 3 songs if more than 5 are passed (defensive)

Error handling:
  - Wrap API call in try/except anthropic.APIError
  - Log exception type and message at WARNING level
  - Return fallback string -- app never crashes due to API issues
